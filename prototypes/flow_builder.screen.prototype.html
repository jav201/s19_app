How to read it

  • Wide regime adds columns: inputs (config/doc), Δbytes, and ranges after — the running memory footprint per block, so you watch it evolve down the pipeline. The right pane adds the run counts, the full check table, and produced artifacts.
  • Status chip: ✓ ok · ◇ notices (notify, don’t block) · ✗ failed · ·· skipped.
  • Twin memory ribbon (wider = finer): █ mapped · ░ gap · █ CRC tail. The before bar shows the pre-CRC footprint so growth is seen.
  • Flow banner: CLEAN ISSUES FAILED — ISSUES = completed-with-issues.
Responsive, one direction. This is the SAME Direction A at two widths: wide unfolds the detail columns + right pane; at the width-narrow floor (<120 cols) they collapse to the vertical-only view. Colors are the app’s live tokens. CRC + Save/Load carry batch-51/batch-52 tags (shown for the vision; batch-50 ships LOAD-notices + CHECK + the status model + this surface). Every ref/finding/path renders markup=False via safe_text.
